# 通用网站自动化场景

本文件提供适用于大多数网站的通用自动化经验，基于小红书等平台的实战总结。

## 核心原则

### 原则1：截图优先原则
**每次操作后必须截图确认结果。**

原因：
- 验证操作是否成功
- 便于调试和问题排查
- 记录操作历史

```bash
# 操作前截图
element.py --screenshot /tmp/before.png

# 执行操作
element.py --click X Y

# 操作后截图（必须）
element.py --screenshot /tmp/after.png
```

### 原则2：动态定位优先
**优先使用 CDP 动态查找元素位置，而非固定坐标。**

固定坐标的问题：
- 不同分辨率下位置变化
- 页面布局变化导致失效
- 响应式设计适配不同屏幕

推荐做法：
- 使用 CDP Runtime.evaluate 执行 JavaScript 查找元素
- 通过 getBoundingClientRect() 获取元素位置
- 基于实际位置执行点击

### 原则3：输入框焦点原则
**输入文本前，必须先确保输入框已获得焦点。**

失败模式：
- 直接调用 --type 输入文本，可能失败

正确模式：
- 先点击输入框获取焦点
- 再输入文本

### 原则4：协议复选框原则
**部分平台需要先勾选"同意协议"复选框，才能点击获取验证码。**

典型平台：
- 小红书：必须先勾选"我已阅读并同意"
- 知乎：自动同意，无需勾选

## 通用登录流程

### 手机号验证码登录标准流程

1. **启动浏览器** 打开目标网站
2. **截图确认** 页面加载完成
3. **点击登录入口** 动态查找登录按钮位置并点击
4. **输入手机号** 先点击输入框获取焦点，再输入
5. **勾选协议**（如需要）部分平台需要先勾选同意协议
6. **点击获取验证码** 动态查找按钮位置并点击
7. **截图确认** 验证码发送状态

## 平台特征

| 特征 | 说明 |
|------|------|
| 元素查找 | --find 对动态/复杂页面经常失效，建议用 CDP 动态查找 |
| 坐标点击 | 可靠，但优先使用动态定位 |
| 文本输入 | 必须先点击输入框获取焦点 |
| CDP 截图 | 对简单页面有效，复杂页面可能失败 |
| Xvfb 截图 | 更可靠，截取整个虚拟屏幕 |

## 已知陷阱

### 陷阱1：元素查找失效
**现象：** --find 返回"未找到匹配元素"
**原因：** 页面使用动态渲染（React/Vue/Angular），元素文本在 DOM 中不存在
**解决：** 使用 CDP Runtime.evaluate 动态查找

### 陷阱2：点击失败
**现象：** 点击元素没有反应
**原因：** 网站有反自动化检测，或元素位置变化
**解决：** 
1. 告知用户"点击失败，尝试使用JavaScript点击..."
2. 使用 `--js-click` 参数重试
3. 如果仍然失败，告知用户"JavaScript点击也失败了，尝试其他方法..."
4. 可以尝试其他方法（如动态查找元素位置后点击）
5. 如果所有方法都失败，告知用户"操作失败，请稍后重试"

### 陷阱3：输入文本无响应
**现象：** --type 执行后页面无变化
**原因：** 输入框未获得焦点
**解决：** 先点击输入框获取焦点，再输入文本

### 陷阱4：获取验证码无响应
**现象：** 点击获取验证码按钮无反应
**原因：** 未勾选"同意协议"复选框（部分平台）
**解决：** 先查找并勾选协议复选框

### 陷阱5：CDP 截图空白
**现象：** 截图文件存在但内容空白或不全
**原因：** 页面使用 Canvas/WebGL 渲染
**解决：** 使用 Xvfb 截图代替

## 使用建议

1. **始终以截图结束** - 验证操作结果
2. **动态定位优先** - 避免固定坐标
3. **输入前点输入框** - 确保获得焦点
4. **检查协议复选框** - 部分平台需要手动勾选
5. **准备 Xvfb 备用** - CDP 失败时切换到虚拟屏幕截图
